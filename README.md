## Project Overview

This is **Texon's first frontend project**, aimed at creating a **user-friendly online bookstore**.  
The project focuses on:

- Clean and minimalist UI
- Easy navigation for books and stationery items
- Consistent typography and color palette
- Reusable components for future Texon projects

---

## Purpose

The goal of this project is to provide a **professional, visually appealing, and functional online store** that can scale with additional products and features.  
It serves as a foundation for all future frontend projects at Texon, ensuring:

- Consistency in design and code
- High-quality user experience
- Ease of maintenance and extension

## Design System

-> Atomic Design

### Colors

| Color       | Hex     | Usage                            |
| ----------- | ------- | -------------------------------- |
| White       | #FFFFFF | Backgrounds, whitespace          |
| Main Accent | #CEB597 | Highlights, buttons, links       |
| Dark Accent | #724742 | Secondary emphasis, headers      |
| Secondary   | #B39A7E | UI elements, borders, cards      |
| Main Text   | #3C3635 | Paragraphs, labels, default text |

### Fonts

| Type | Font       | Usage                       |
| ---- | ---------- | --------------------------- |
| Main | Aref Ruqaa | Headings, titles            |
| Body | Tajawal    | Paragraphs, labels, UI text |

> The design system ensures **typography, spacing, and colors** are consistent across the entire project.

---

> All components should be **reusable** and follow the design system for consistency.

## Usage Guidelines

- Follow the **design system** for all UI elements.
- Use **reusable components** wherever possible.
- Maintain **clean code principles** and consistent naming conventions.
- Keep the interface **minimalist and user-friendly**.

## Contributing

- Follow clean code and consistent naming conventions.
- Keep **spacing, colors, and typography** aligned with the design system.
- Write **clear and meaningful commit messages**.
- Ensure all new components and pages maintain **reusability** and design consistency.
